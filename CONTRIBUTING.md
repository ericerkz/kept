## Development Process

1. Open an issue before making large changes.
2. Keep pull requests focused and reasonably small.
3. Include tests where practical.
4. Do not submit code copied from another project unless the license is compatible
   and the original notice is preserved.
5. Do not submit AI-generated or third-party-generated code unless you have
   reviewed it and have the right to contribute it under this agreement.
6. Make sure your contribution does not introduce dependencies with licenses that
   conflict with AGPLv3 or Kept's commercial licensing goals.
